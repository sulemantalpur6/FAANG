## Pattern 1: Two Pointers
**Concept**: Use two pointers to traverse array/string from both ends or same direction.
**When to use**: Sorted arrays, palindrome checks, in-place modifications, merging.

- [ ] [26. Remove Duplicates from Sorted Array](https://leetcode.com/problems/remove-duplicates-from-sorted-array/) - In-place array modification
- [ ] [27. Remove Element](https://leetcode.com/problems/remove-element/) - Two pointer removal
- [ ] [88. Merge Sorted Array](https://leetcode.com/problems/merge-sorted-array/) - Two pointers from end
- [ ] [125. Valid Palindrome](https://leetcode.com/problems/valid-palindrome/) - Two pointers + string check
- [ ] [344. Reverse String](https://leetcode.com/problems/reverse-string/) - Classic two pointer swap
- [ ] [283. Move Zeroes](https://leetcode.com/problems/move-zeroes/) - Partitioning with two pointers
- [ ] [977. Squares of a Sorted Array](https://leetcode.com/problems/squares-of-a-sorted-array/) - Two pointers merge

## Problem-Solving Strategy

**Step 1: Understand (5-10 min)**
- Read problem twice
- Write down inputs/outputs
- Identify edge cases
- Recognize the pattern

**Step 2: Plan (5-10 min)**
- Think of brute force first
- Write pseudocode
- Don't code yet!

**Step 3: Code (15-20 min)**
- Implement your plan
- Test with examples

**Step 4: Optimize (10 min)**
- Reduce time/space complexity

**Time Limit**: 30-40 minutes per problem initially
